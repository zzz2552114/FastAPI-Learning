## Form 表单数据与 JSON 的对比

### 1. 数据格式与结构

#### Form 表单数据
结构扁平，由一系列“字段名=字段值”组成，多个字段间用 `&` 连接。

**示例：**
提交用户名、年龄和爱好，其请求体为：
```
username=John&age=25&hobbies=reading&hobbies=coding
```
可以看到，多个爱好（`hobbies`）只能通过重复字段名来传递，结构不直观。

#### JSON
结构清晰，支持嵌套对象和数组，能完美映射复杂的数据模型。

**示例：**
同样的数据，JSON 的表示方式是：
```json
{
  "username": "John",
  "age": 25,
  "hobbies": ["reading", "coding"]
}
```
这种结构一目了然，更易于程序解析和维护。

---

### 2. 典型应用场景

| 数据格式          | 应用场景                                                                  |
|:--------------|:----------------------------------------------------------------------|
| **Form 表单数据** | 主要用于传统网页的交互。当你在浏览器中提交一个 HTML 表单时，浏览器默认就会以这种格式发送数据。                    |
| **JSON**      | 是现代 API 开发的事实标准。在前后端分离、移动端 App 等场景中，需要传输更复杂的数据，JSON 的灵活性和结构化特性使其成为首选。 |

---

### 3. 在 FastAPI 中的使用区别

#### 声明方式
- **接收 JSON 数据**：直接使用 Pydantic 模型作为参数即可。
- **接收 Form 数据**：必须显式地使用 `Form()` 函数。如果遗漏 `Form()`，FastAPI 会将其误认为是查询参数或 JSON 请求体。

#### 互斥性
在一个路径操作中，**不能同时声明 `Form` 参数和 JSON `Body` 参数**。因为一个请求只能使用一种 `Content-Type`（`application/x-www-form-urlencoded` 或 `application/json`），这是 HTTP 协议的规定。

#### 数据模型限制
由于 Form 数据的扁平特性，你**不能直接将 `Form` 参数声明为 Pydantic 模型**。每个表单字段都需要单独使用 `Form()` 进行声明。
